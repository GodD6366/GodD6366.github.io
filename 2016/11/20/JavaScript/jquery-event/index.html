jQuery中事件的绑定

发表于 2016-11-20 | 分类于 jQuery | | 阅读次数:

Event Handler Attachment

.on() 方法

两种参数形式：

.on( events [, selector ][, data ], handler )

events: 一个或多个可选的事件和命名空间，比如：”click” or “keydown.myPlugin”

selector: 一个选择器，用来匹配后代的元素，如果选择器为空或者没写，事件同样会在选择的元素（绑定的元素）上执行

data: 当一个事件被触发时，要传递给事件处理函数的 event.data

handler: 事件执行时所需要使用的函数,此函数可以 return false 并且可以简写成 false

.on( events [, selector ][, data ] )

events: 这种形式的 event 是一个对象，同样可以绑定一个或多个事，但是采用键值对的形式，键后面可以跟需要执行函数

selector: _同上_

data: _同上_

Event names and namespaces

事件名称只能使用：字母、下划线、冒号

调用.trigger()可以模拟用户操作，执行事件

命名空间的使用是为了在取消绑定一个事件的时候不影响其他相同的事件，

如：”click.myPlugin.simple”，卸载可以通过.off(“click.myPlugin”)或者.off(“click.simple”)或者.off(“click.myPlugin.simple”)，
命名空间的使用类似 CSS，但是不分层次，而且必须匹配写入的所有命名空间才会卸载该元素的相应事件，比如”click.myPlugin.simple”只会卸载同时包含.myPlugin 和.simple 的元素的 click 事件，如果元素仅在.myPlugin 的命名空间，该元素的 click 事件不会被卸载！

Direct and delegated events

因为冒泡机制，在传播路径上的所有绑定相同事件的元素都会被执行

选择器为 null 或者 undefined 的时候，事件会直接绑定在当前元素上，选择器有值的时候，
事件绑定在父辈元素上，当点击子元素的时候，事件会冒泡到父辈元素上，当与选择器匹配的时候就会执行事件，否则不会执行

事件绑定一般情况下是必须绑定已经存在的元素，但是使用 ON 的时候，可以伪绑定（实际不是绑定，是运用冒泡的原理实现代理）还没有存在的元素

The event handler and its environment

handler 函数是必要的，可以使用匿名函数 可以通过调用 trigger()方法进行模拟事件的触发

事件的冒泡可以使用event.stopPropagation()防止事件冒泡

可以使用event.stopImmediatePropagation()保证事件按顺序执行

可以调用的event.preventDefault()阻止浏览器的默认操作

因为event.stopPropagation()和event.preventDefault()默认返回的是false，如果函数里只包含这两个语句，可以简写成\$( “a.disabled” ).on( “click”, false );

1.在直接绑定即无选择器的时候 this 表示当前绑定的元素

2.在代理事件时,this不一定一直等于event.target，this会随着冒泡经过的元素而变化.
this代表当前产生事件冒泡的元素，event.target代表产生该事件的元素

Passing data to the handler

data 是可选的，如果不是 null 或者 undefined，那么处理程序会获得这个值，如果是字符串就必须保证选择器是存在的，否则会存在误读，一般是键值对的形式

Event performance

为了提高性能，绑定事件选择器的时候尽量选择靠近的的父辈元素，不要过多的绑定在 body 和 document 上，一些使用频繁的事件可以用定时器控制刷新频率

Additional notes

一些方法不支持冒泡，如：focus、blur ，jQuery 定义了支持冒泡的 focusin、focusout 代替，使用时建议使用后者

.bind()和.on()的区别

on 比 bind 多了一项功能，当 on 参数列表中没有提供 selector，这时 bind 和 on 功能类似，当 on 有提供 selector 参数，\$set 就被委托了一个 handler 处理满足 selector 的子元素的相应事件

.delegate()和.on()的区别

可以实现和 on 一样的效果，区别在于 delegate 的选择器参数在第一个，on 在第二个

.one()

看源码可以发现，one 就是调用一次 on，然后 off

.trigger()

模拟事件的激活，会进行冒泡，使用.stopPropagation()可以阻止冒泡

.stopPropagation()后会自动return false

.triggerHandler()

.triggerHandler() 方法并不会触发事件的默认行为

.triggerHandler() 仅影响第一个匹配到的元素，并且不会冒泡

如果没有触发任何事件会返回 undefined

.off()

移除事件时，可以命名空间进行移除，可以防止同类型的事件被误移除
